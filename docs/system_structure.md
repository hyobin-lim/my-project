# 🗺️ 시스템 구조도 (System Structure)
*최종 업데이트: 2026-02-24 18:00:20*

이 문서는 '문화생활 통합 플랫폼' 프로젝트의 전체적인 구조와 관리 체계를 설명합니다.

## 1. 프로젝트 폴더 구조 (Directory Map)

```text
my-project/
├── data/               # 데이터 관련 자산
│   ├── agents/         # 4단계 멀티 에이전트 체계
│   │   ├── director/   # [NEW] 전체 조율 및 스케줄링 (총괄)
│   │   ├── collector/  # 데이터 수집 (Raw)
│   │   ├── organizer/  # 데이터 정제 및 분류 (Refined)
│   │   └── manager/    # 데이터 저장 및 배급 (API-Ready)
│   └── store/          # 단계별 데이터 저장소 (raw/ refined/)
├── docs/               # 프로젝트 관리 및 기술 문서
│   ├── vision_roadmap.md    # 장기 계획 및 철학
│   ├── task_tracker.md      # 단기 할 일 및 진행 현황
│   ├── development_log.md   # 일자별 상세 수행 기록
│   ├── troubleshooting.md   # 오류 해결 및 교훈 기록
│   ├── handoff_guide.md     # 학원-집 이동 시 즉시 실행 가이드
│   └── system_structure.md  # [현재 문서] 전체 구조 안내
└── web/                # 리액트 기반 웹 프론트엔드
    ├── src/
    │   ├── components/ # 재사용 가능한 UI 부품
    │   ├── types/      # 데이터 형식 정의 (TypeScript)
    │   ├── App.tsx     # 메인 조립 페이지
    │   └── App.css     # 메인 스타일시트
    └── index.html      # 웹 진입점
```

## 2. 문서 체계 상세 가이드 (Documentation System)

| 문서명 | 주요 내용 | 기록 시점 |
| :--- | :--- | :--- |
| **Vision & Roadmap** | 프로젝트 목표, 핵심 가치, 기술 스택 | 큰 방향성이나 기술이 바뀔 때 |
| **Task Tracker** | 현재 진행 중인 To-Do, 완료된 작업 | 새로운 작업을 시작하거나 마칠 때 |
| **Development Log** | 시간대별 작업 내용, 결정 사항 | 매 작업 세션 종료 후 |
| **Troubleshooting** | 에러 로그, 해결 방법, 계획 변경 이유 | 예상치 못한 문제 해결 시 |
| **Handoff Guide** | 환경 전환 시 즉시 실행할 체크리스트 | 매 세션 종료 전 |
| **System Structure** | 폴더 구조, 문서 역할, 시스템 설계 | 구조적인 변화가 생길 때 |

## 3. 카테고리 분류 체계 (Taxonomy Guide)
*2026-02-25 확정*

모든 데이터는 '무료 문화생활 내비게이터'의 정체성에 따라 아래 체계로 분류 및 관리됩니다.

### **[L1: 메인 카테고리 정의]**
1. 🎭 **공연**: 정해진 시간에 특정 장소에서 관람하는 예술 활동.
2. 🎬 **영화**: 시사회, 무료 상영 등 스크린을 통한 시청 활동.
3. 🖼️ **전시**: 자유로운 시간에 방문하여 관람하는 시각 예술 활동.
4. 🎡 **축제/체험**: 사용자가 직접 참여하고 활동하는 문화 행사 및 체험.
5. 🎓 **교육/강연**: 지식 습득 및 자기계발을 위한 유익한 배움 활동.

### **[L2: 🎭 공연 (Performance) 세부 분류]**
| 세부 분류 (L2) | **L3 (세부 항목/포함 범위)** | 내비게이터 팁 (Curation Point) |
| :--- | :--- | :--- |
| **연극/뮤지컬** | 연극, 뮤지컬 갈라쇼, 낭독극, 학생/아마추어 연극 | '극단 지원금' 유무 확인 필수 |
| **클래식/오페라/무용** | 독주/협주회, 오케스트라, 합창/성악, 발레, 현대무용, 오페라 | 제목에 장르/악기 태그 필수 |
| **국악/전통예술** | 국악/판소리, 사물놀이/풍물, **탈춤/마당놀이**, 전통무용 | 공공기관 정기 무료 상연 강조 |
| **대중음악** | 인디/재즈/록, 야외 콘서트, 축제 공연, 라이브 클럽 | '음료 주문 필수' 여부 가이드 |
| **버스킹/거리공연** | 거리 음악(버스킹), 마임/서커스, 거리 퍼포먼스 | 날씨 및 장소 정보가 핵심 |
| **아동/어린이/마술** | 인형극, **아동/어린이 뮤지컬**, 마술쇼, 구연동화 | 권장 연령 및 편의시설 가이드 |
| **기타** | 토크 콘서트, 낭독회, 복합 예술 퍼포먼스 | 실험적/복합 장르 안내 |

### **[L2: 🎬 영화 (Cinema) 세부 분류]**
| 세부 분류 (L2) | **L3 (세부 항목/실무 경로)** | 내비게이터 팁 (Curation Point) |
| :--- | :--- | :--- |
| **시사회/혜택** | 시사회 응모, 무대인사(무료), 선착순 예매권, 제휴 혜택/쿠폰, 극장 이벤트 | **응모 페이지 직링크** 및 마감일 고지 |
| **야외/자동차 상영** | **자동차 극장(무료)**, 한강 무비나잇, 공원/숲속 상영, 옥상 영화관, 찾아가는 영화관 | **장소가 주는 낭만**과 특별한 경험 강조 |
| **아동/애니메이션** | 어린이 전용 상영, 아동용 애니메이션, 구연동화 영화 | 부모님들을 위한 **아이 동반 환경** 정보 |
| **마을/공공 상영** | 도서관 주말 영화, 미디어센터, 구청 강당, 주민센터 상영 | 전 연령 대상 **우리 동네** 무료 영화 |
| **영화제/이벤트 상영** | 국제/국내 영화제, **축제 연계 상영**, 팝업 시네마, 브랜드 기획전 | **축제 속 특별한 상영** 및 이벤트성 정보 |
| **시네마 토크/교육** | **감독/배우 GV**, 영화 해설/강연, 영화 제작 체험, 영상 장비 워크숍 | **영화를 깊게 즐기는 법** 및 배움 위주 |
| **예술/고전/특수** | 영상자료원, 독립/예술영화, 고전 복원작, 명작 기획전, 실버/배리어프리 | **희소성 있는 명작** 및 특정 대상 안내 |

## 4. 데이터 파이프라인 및 수집 전략 (Data Strategy)

### **[비용 투명성 필터 (Cost Transparency)]**
사용자가 정보를 즉각적으로 판단할 수 있도록 아래 기준으로 정보를 투명하게 고지함 (L3 진입 시 필터 제공):
- **[100% 무료]**: 아무런 비용 발생 없음.
- **[소액 지원금]**: 극단 발전을 위한 5,000원 내외의 지원금 등.
- **[음료 주문 필수]**: 입장료는 없으나 1인 1음료 주문 필수.
- **[사전 예약제]**: 비용은 없으나 사전 예약 시스템 이용 필수.

## 5. 핵심 개발 원칙 (Core Principles)
1. **Web First**: 데이터 수집보다 눈에 보이는 웹 UI를 먼저 구축하여 가시성 확보.
2. **Component Based**: 모든 UI는 독립된 부품(Component)으로 만들어 재사용성과 관리 효율을 높임.
3. **Double Check**: 위험한 작업(삭제, 리셋 등) 전에는 반드시 파트너와 상의 후 진행.
4. **Honesty & Limits (Absolute)**: AI는 자신의 한계를 인정하고 모르는 것을 추측하지 않으며, 파트너와 상호 정직하게 소통함. (2026-02-24 확립)
5. **Efficient Role Division**: 대용량 다운로드, 패키지 설치, 상호작용이 필요한 명령어는 파트너가 직접 실행하여 속도와 안정성을 확보함.
6. **Clean Docs**: 모든 작업 결과와 시행착오는 약속된 문서 체계에 정확한 시스템 시간 확인 후 즉시 기록함.
