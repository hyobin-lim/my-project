# 🛠️ 트러블슈팅 및 교훈 (Troubleshooting & Lessons)
*최종 업데이트: 2026-02-28 21:00:00*

## 1. 명령어 환경 관련 (Windows PowerShell)
- **문제**: `&&` 연산자를 사용한 연속 명령어 실행 실패.
- **원인**: PowerShell 설정에 따라 `&&` 인식이 안 될 수 있음.
- **해결**: 세미콜론(`;`)을 사용하거나 명령어를 개별적으로 실행함.

## 2. 프로젝트 전략 변경
- **상황**: 초기 계획은 데이터 수집 우선이었으나, 가시성을 위해 웹 우선 개발로 변경.
- **이유**: 결과물을 눈으로 확인하며 개발할 때 동기부여와 설계 정밀도가 높음.

## 3. 도구 활용 및 설치
- **문제**: 무거운 설치 작업이나 상호작용이 필요한 명령어에서의 타임아웃 및 오류.
- **해결**: 복잡한 환경 설정은 파트너가 직접 수행하고, AI는 이를 지원하는 역할 분담 체계 확립.

## 4. 환경 인코딩 및 파일 관리
- **문제**: 파일명 한글화 과정에서 인코딩 깨짐 현상 발생.
- **해결**: UTF-8 인코딩을 준수하고, 깃(Git) 한글 깨짐 방지 설정(`quotepath false`) 적용.

## 6. 디자인 강박과 데이터 왜곡
- **문제**: AI가 메가 메뉴의 시각적 대칭을 위해 데이터 밀도와 상관없이 카테고리 개수를 7개로 강제 설정함.
- **원인**: '심미적 완성도'를 '데이터의 진실성'보다 우선시한 AI의 고질적 편향.
- **해결**: 대칭성을 과감히 버리고 실제 수집 가능한 데이터 비중에 따라 카테고리를 재구성함 (Data-Driven Taxonomy).
- **교훈**: 기초를 다지는 단계에서는 모양보다 '데이터의 논리적 구조'가 훨씬 중요하다는 점을 명심함.

## 11. [2026-02-28] 세션 중단 및 데이터 유실 사고와 복구
- **문제 1 (논리 단절)**: 파트너와의 치열한 토론 중 세션이 끊겨 합의된 설계 로직(L3/L4/L5)이 휘발됨. AI가 재시작 시 이를 기억하지 못해 논의가 도돌이표가 됨.
- **문제 2 (데이터 유실)**: AI가 신규 분석 데이터를 추가하는 과정에서 파일 전체를 덮어씌워 기존의 방대한 분석 기록을 삭제함.
- **해결 1 (LOGIC_PERSISTENCE)**: 루트 디렉토리에 **`LOGIC_PERSISTENCE.md`**를 도입. 토론 중 합의된 '로직의 핵심'을 즉시 박제하고, 매 세션 시작 시 최우선으로 읽도록 프로토콜(PARTNERSHIP.md)을 강화함.
- **해결 2 (Git Recovery)**: 유실된 데이터는 `git log -p`를 통해 이전 커밋 기록에서 텍스트를 추출하여 완벽하게 복원함.
- **교훈**: "문서화되지 않은 로직은 존재하지 않는 것이다." 특히 설계 단계에서의 구두 합의는 즉시 시스템(문서)에 이식해야 '논리적 연속성'을 지킬 수 있음.
