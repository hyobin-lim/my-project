# 🛠️ 트러블슈팅 및 교훈 (Troubleshooting & Lessons)
*최종 업데이트: 2026-02-24 17:46:13*

## 1. 명령어 환경 관련 (Windows PowerShell)
- **문제**: `&&` 연산자를 사용한 연속 명령어 실행 실패.
- **원인**: PowerShell 버전이나 설정에 따라 `&&` 인식이 안 될 수 있음.
- **해결**: 세미콜론(`;`)을 사용하거나 명령어를 개별적으로 실행함.

- **문제**: `touch` 명령어 인식 불가.
- **원인**: 윈도우 기본 명령어가 아님.
- **해결**: `New-Item` 명령어를 쓰거나 에이전트가 직접 `write_file`을 사용하여 파일 생성.

## 2. 프로젝트 전략 변경
- **상황**: 초기 계획은 데이터 수집(Collector) 우선이었으나, 가시적인 결과물을 위해 웹(Frontend) 우선 개발로 변경.
- **이유**: 데이터만 쌓이는 것보다 눈으로 확인 가능한 UI가 있을 때 개발 동기부여와 효율성이 높음. (파트너 제안 반영)

## 3. 도구 활용 (Vite)
- **문제**: `npx create-vite` 실행 시 상호작용 질문(Vite 8 beta 등)으로 인한 타임아웃.
- **해결**: `--template react-ts --yes` 옵션 사용 혹은 **파트너 직접 실행**으로 전환.
- **교훈**: 설치 및 초기화와 같은 무거운 작업은 파트너가 직접 수행하는 것이 훨씬 빠르고 안정적임.

## 4. 협업 태도 및 소통
- **문제**: AI가 한계를 넘어서는 작업을 무리하게 시도하거나 추측하여 정보를 제공하는 현상.
- **해결**: '모르는 것은 모른다고 말하기', '무리하게 진행하지 않기'를 절대 원칙으로 세움.
- **결과**: 파트너와의 신뢰를 유지하고 돌이킬 수 없는 오류를 방지함.
