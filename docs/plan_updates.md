# 프로젝트 계획 및 전략적 결정 (Plan Updates)

## 2026-02-24 16:30:00 - 시스템 구조 확정 및 파트너십 원칙

### 1. 프로젝트 개요
- **목표**: 문화생활 통합 플랫폼 개발 (웹 & 앱) 및 후원 모델 구축.
- **핵심 가치**: 흩어진 무료 문화 정보를 한곳에 모아 제공하는 '원스톱 서비스'.
- **협업 방식**: 파트너(사용자)와 AI가 동등한 입장에서 서로 돕고 가르치는 구조.

### 2. 핵심 전략
1. **웹 우선 개발 (Web First)**: 데이터 수집보다 수집된 데이터를 보여줄 웹 프론트엔드 환경을 먼저 구축 (2026-02-24 결정).
2. **모바일 퍼스트 및 컴포넌트 기반 설계**: 재사용 가능한 UI 부품(Component) 중심의 아키텍처 도입으로 개발 효율성 극대화.
3. **데이터 안전**: 구글 드라이브 자동 백업 및 위험 작업 전 상호 확인.
4. **천천히 차근차근**: 속도보다 기초와 원리 이해를 우선시하는 개발 방식.

### 3. 프로젝트 폴더 구조 (계층형)
- `docs/`: 기획 및 개발 기록 문서
- `data/bot/`: 파이썬 기반 데이터 수집 로봇
- `data/store/`: 수집된 JSON 데이터 저장소
- `web/`: 리액트 기반 웹 서비스 소스 (React + Vite 6)

### 4. 기술 스택
- **Language**: Python (Collector), JavaScript/TypeScript (Web).
- **Framework**: React (Vite 6), TypeScript.
- **Automation**: GitHub Actions.
- **AI**: Google Gemini API.
