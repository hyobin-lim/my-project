# 💎 FREEISM: THE CREED

> **지갑은 [ , ] 영감은 [ ! ]**
> 
> **FREEISM** = **Free**(무료) + **Ism**(철학) + **Prism**(굴절/연결)
>
> 우리는 단순한 정보 제공자가 아닙니다. 흩어져 있는 '무료'라는 원석(Raw Data)을 발견하여, 우리만의 엄격한 철학(Ism)이라는 프리즘을 통해 다채로운 문화 예술의 빛으로 사용자에게 전달하는 **전문 중개인 시스템**입니다.
> 
> - **지갑은 [ , ]**: 경제적 부담은 쉼표처럼 잠시 내려놓고 쉬어가는 곳.
> - **영감은 [ ! ]**: 문화적 자극과 가치는 느낌표처럼 강렬하게 채워가는 곳.
> - **우리는 소유하지 않고 연결하며, 숨겨진 가치를 굴절시켜 증폭시킵니다.**

## 🏗️ 1. 하이엔드 나무 모델 (High-End Tree Model v2)
현재 시스템의 근간이 되는 최신 설계 논리이며, 장르 중심의 분류를 탈피하여 '사용자의 욕망 동선'과 '기회의 등급'을 결합한 유연한 6단계 모델이다.

### **계층별 역할 정의 (L1-L6)**

- **[L1: 뿌리 (Root / Domain)]**
    - **정의**: 서비스의 기반이 되는 거대 도메인. 모든 데이터가 영양분을 공급받는 시작점.
    - **역할**: 서비스의 핵심 정체성을 규정하며, 수집되는 모든 문화 데이터를 6대 도메인으로 분류함.

**상세 정의 (6대 도메인)**:
    1. 🎭 **공연**: 정해진 시간에 특정 장소에서 관람하는 예술 활동.
    2. 🎬 **영화**: 시사회, 무료 상영 등 스크린을 통한 시청 활동.
    3. 🖼️ **전시**: 자유로운 시간에 방문하여 관람하는 시각 예술 활동.
    4. 📺 **방송(방청/직관)**: 방송국 스튜디오 등에서 이루어지는 음악방송, 토크쇼 등의 방청 및 직관 체험.
    5. 🎪 **축제/체험**: 사용자가 직접 참여하고 활동하는 문화 행사 (주인 데이터).
    6. 🎓 **교육/강연**: 지식 습득 및 자기계발을 위한 유익한 배움 활동.

- **[L2: 기둥 (Trunk / 탐색 방향)]**
    - **정의**: 대분류를 지탱하는 핵심 기둥. 나무의 중심을 잡아주는 굵은 줄기.
    - **역할**: 사용자의 현재 상황과 거대한 취향의 줄기를 결정. 장소(실내/야외)와 획득 전략의 성격이 결합된 탐색의 첫 관문. (예: "이야기에 몰입하고 싶어", "나 연극 보고 싶어!").
- **[L3: 굵은 나뭇가지 (Major Branch / 취향 응집)]**
    - **정의**: 기둥에서 뻗어 나온, 비슷한 성격과 취향의 데이터들이 응집력 있게 묶인 핵심 줄기.
    - **역할**: 사용자의 막연한 취향을 실질적인 데이터 뭉텅이로 연결하여 인지 부하를 줄임. 단순히 데이터를 쪼개는 것이 아니라, 비슷한 것끼리 단단하게 뭉쳐두어 L4-L5까지 한눈에 도달하게 함. (예: 대학로 상설 연극, 클래식/오페라, 인디/재즈).
- **[L4: 잔가지 (Twig / 세부 확정)]**
    - **역할**: 응집된 취향 안에서 자신의 목적지를 정밀하게 타겟팅. 연극/어린이 섹션의 확정 항목들을 포함하며, L5로 넘기기 전 최종 분류 수행. (예: 프리미엄 대형 작품, 캐릭터 뮤지컬, 오늘 밤 버스킹).
- **[L5: 방법론 (Methodology / Action Engine)] [핵심 엔진]**
    - **정의**: 데이터를 획득하는 실질적인 '방법'. 분류 체계의 가장 중요한 차별점이자 핵심 엔진.
    - **역할**: **'어떻게 0원에 즐길 것인가'**에 대한 실질적인 행동 전략(전략적 필살기) 제안 단계. 이 선택 이후 국면이 전환되어 결실(L6)이 맺힘.
- **[L6: 이파리/열매 (Leaf / Fruit / Result)] [전용 뷰]**
    - **정의**: L5의 전략 선택 시 **새로운 창(View)으로 열리는** 풍성한 데이터 세계. 사용자 탐색의 종착지.
    - **역할**: L3-L5의 정교한 필터링 결과가 집약된 독립적이고 가치 있는 큐레이션 공간.

## 💎 3. 기회 등급 체계 (Grade S~D Logic)
사용자가 무료 문화를 획득하는 데 드는 노력과 기회의 가치를 계층화한 엔진.

| 등급 (Grade) | 성격 (Nature) | 획득 방법론 (L5 Example) | 예시 콘텐츠 |
| :--- | :--- | :--- | :--- |
| **Grade S (Premium)** | 로또급 가치 / 고가 대작 | 초대권 응모 (당첨 대기) | 대형 뮤지컬, 국립단 정기공연 |
| **Grade A (Mainstream)** | 노력형 획득 / 확실한 퀄리티 | 사전 예약 (피케팅) | 서울스테이지11, 대학로 인기연극 |
| **Grade B (Casual)** | 현장성 / 즉시 향유 | 현장 선착순 / 자유 관람 | 광장 버스킹, 박물관 로비음악회 |
| **Grade C (Community)** | 나눔 / 교육 / 아마추어 | 상시 신청 / 선착순 | 구민회관 공연, 학생/시민 공연 |
| **Grade D (Alternative)** | 유료 전환 / 할인 혜택 | 외부 사이트 연동 (타임티켓 등) | 잔여석 급매, 정가 대비 할인가 |

## 💻 4. 구현 원칙

- **데이터 실재론 기반 비대칭 리팩토링**: 디자인적 대칭 강박을 버리고 실제 데이터 비중에 UI를 맞춘다.
- **주인(Host) vs 손님(Guest) 논리**: 
    - **주인 데이터**: 행사 자체가 목적인 카테고리 (예: 영화제, 재즈 페스티벌) -> [축제] 도메인 소속.
    - **손님 데이터**: 축제 내 찬조 출연 콘텐츠 (예: 벚꽃 축제 내 영화 상영) -> 해당 장르 도메인 내 `축제/이벤트 연계`로 배치.
