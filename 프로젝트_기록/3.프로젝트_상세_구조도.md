# 💎 FREEISM: THE CREED

> **지갑은 [ , ] 영감은 [ ! ]**
> 
> **FREEISM** = **Free**(무료) + **Ism**(철학) + **Prism**(굴절/연결)
>
> 우리는 단순한 정보 제공자가 아닙니다. 흩어져 있는 '무료'라는 원석(Raw Data)을 발견하여, 우리만의 엄격한 철학(Ism)이라는 프리즘을 통해 다채로운 문화 예술의 빛으로 사용자에게 전달하는 **전문 중개인 시스템**입니다.

## 🏗️ 1. 하이엔드 나무 모델 (High-End Tree Model L1-L6)
사용자의 욕망 동선과 기회의 등급을 결합한 6단계 큐레이션 엔진입니다.

- **[L1: 뿌리 (Domain)]**: 6대 핵심 도메인 (공연, 영화, 전시, 방송, 축제, 교육).
- **[L2: 기둥 (Trunk)]**: 장소(실내/야외) 및 장르적 취향의 큰 줄기.
- **[L3: 굵은 나뭇가지 (Branch)]**: 비슷한 성격의 데이터 응집 (예: 대학로 연극, 클래식).
- **[L4: 잔가지 (Twig)]**: 세부 목적지 타겟팅 (예: 프리미엄 대형 작품, 오늘 밤 버스킹).
- **[L5: 방법론 (Methodology)] [핵심 엔진]**: **'어떻게 0원에 즐길 것인가'** 전략 선택 (Grade S-D).
- **[L6: 이파리/열매 (Leaf/Fruit)] [전용 뷰]**: L5 선택 시 열리는 독립적이고 풍성한 데이터 큐레이션 결과물.

## 💎 2. 기회 등급 체계 (Free Grading System)
데이터 실재론에 기반한 0원 획득 가치 등급입니다.

- **Grade S (Jackpot)**: 순수 무료 / 로또급 초대권 응모 (예: 대형 뮤지컬, 국립단 초대권).
- **Grade A (Mainstream)**: 노력형 획득 / 확실한 퀄리티 (예: 서울스테이지11, 사전 예약 피케팅).
- **Grade B (Casual)**: 현장 선착순 / 만원의 벽(10,000원 이하) 이하 casual 체험.
- **Grade C (Community)**: 지역 기반 나눔 / 교육 / 아마추어 공연.
- **Grade D (Alternative)**: 유료 전환 대비 탈출구 전략 (예: 잔여석 급매, 할인 연동).

## 📊 3. 6대 도메인별 기둥 및 나뭇가지 (Trunk & Branch)
*실제 데이터 밀도에 기반하여 확정된 큐레이션 경로입니다.*

### **1) 🎭 공연 (Performance) [Masterpiece Target]**
- **연극/뮤지컬**: 프리미엄/대형, 대학로/상설, 학생/시민/아마추어
- **클래식/무용**: 독주/교향악, 오페라, 발레/무용, 전통 무용
- **국악/대중음악**: 판소리/민속, 인디/재즈, K-POP/쇼케이스, 거리공연(버스킹)
- **어린이/가족**: 캐릭터 뮤지컬, 가족 음악회, 참여형 퍼포먼스

### **2) 🎬 영화 (Cinema)**
- **시사회/초대**: 응모형 시사회, 통신사/기업 우대 혜택
- **공공 상영**: 도서관, 미디어센터, 박물관/미술관 상영
- **독립/예술**: 독립영화, 시네마테크 기획전, GV/영화 교육

### **3) 🖼️ 전시 (Exhibition)**
- **기획/특별**: 대형 기획전, 브랜드 팝업, 축제 연계 특별전
- **공공/상설**: 국공립 미술관/박물관, 대학 갤러리, 외국문화원
- **스트릿/야외**: 공공미술, 미디어 파사드, 벽화마을/아트로드

### **4) 📺 방송 (Broadcast)**
- **음악/쇼**: 음악 방송 방청, 시상식 직관, 공개 코미디
- **교양/라디오**: 강연/토론 방청, 라디오 공개방송

### **5) 🎪 축제 (Festival)**
- **문화/예술**: 영화제, 공연 페스티벌, 비엔날레/아트페어
- **자연/역사**: 꽃/생태 축제, 고궁/문화재 야행, 지역 특산물 축제

### **6) 🎓 교육 (Education)**
- **인문/창작**: 북토크, 예술 강연, 메이커/취미 공예
- **디지털/미래**: 코딩, AI/데이터, 유투브/미디어 제작

## 🏛️ 4. 데이터 실재론 운영 원칙
1. **무손실 원칙 (Never-Omit)**: 모든 리터럴 데이터는 100% 보존하며 생략을 금지한다.
2. **데이터 밀도 우선**: 디자인적 대칭보다 실제 데이터가 존재하는 영역에 UI 에너지를 집중한다.
3. **주인 vs 손님**: 행사 자체가 목적인 '주인'과 찬조 출연인 '손님' 데이터를 명확히 구분하여 배치한다.
