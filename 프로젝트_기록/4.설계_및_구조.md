# 🏗️ 4. 설계 및 구조 (시스템/웹 구현)
*이 문서는 프로젝트의 기술 스택, 폴더 구조, 컴포넌트 설계 등 실제 개발 구현을 위한 **'시스템 설계 도면'**입니다.*
*최종 업데이트: 2026-03-06 (Vite 6 + React 19 환경 기반 실구현 완료)*

---

## 🛠️ 1. 기술 스택 (Tech Stack)
- **Frontend**: React 19, Vite 6, TypeScript, Vanilla CSS.
- **Backend/Collector**: Python 3.14 (.venv), requests, BeautifulSoup.
- **Data Storage**: JSON 기반 파일 스토리지 (`store/raw`, `store/refined`).

---

## 📂 2. 프로젝트 폴더 구조 (System Directory Tree)

```text
my-project/
├── .venv/                      # Python 가상 환경
├── 프로젝트_기록/              # 핵심 설계 및 로그 문서
├── data/                       # [예정] 데이터 수집 및 가공 엔진
└── web/                        # React 프론트엔드 (Vite 6)
    ├── public/                 # 정적 자산

        ├── components/
        │   └── organisms/      # Header.tsx (단계적 레이어 GNB)
        ├── constants/
        │   └── categories/     # 도메인별 모듈화 (performance.ts 등)
        ├── pages/
        │   └── Home.tsx        # 여기어때 스타일 큐레이션 홈
        ├── types/              # culture.ts (L1-L6 인터페이스 정의)
        ├── App.tsx             # 루트 컴포넌트 및 기본 스타일
        ├── index.css           # 글로벌 스타일 (데스크탑 와이드 최적화)
        └── main.tsx            # 엔트리 포인트
```

## 🏛️ 3. 핵심 UI/UX 엔진 설계

### **1) GNB: 단계적 레이어 내비게이션 (Hover-Depth Reveal)**
- **철학**: 정보의 깊이는 유지하되 시각적 난잡함을 제거한다.
- **구조**: L1 호버 시 L2/L3 메가 메뉴 전개 -> 특정 L3 호버 시 L4(잔가지)가 칩(Tag) 형태로 우아하게 나타남.

### **2) 홈 화면: 비주얼 블록 큐레이션**
- **벤치마킹**: '여기어때' 스타일의 수평 스크롤 카드 레이아웃.
- **섹션 전략**: [🔥 인기 S급 / ⏰ 마감임박 / 👶 가족 추천] 등 데이터 밀도와 유저 욕망 중심의 블록화 배치.

### **3) 개발 표준**
- **철저한 모듈화**: 6대 도메인별 카테고리 독립 모듈 관리 (`src/constants/categories/`).
- **데스크탑 와이드 우선**: 1280px 해상도 최적화 레이아웃 적용.
- **타입 안정성**: 하이엔드 나무 모델 전용 인터페이스(`CategoryItem`, `CultureDomain`)를 통한 엄격한 타입 관리.
