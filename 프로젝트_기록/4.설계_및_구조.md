## 📅 [2026-03-08 14:20:00] [누적] 이원화 아키텍처 및 모듈화 사령부 설계 (Supreme V10.0)

### **🏗️ 1. 이원화된 세계 (World Separation Architecture)**
- **Service Domain (Web)**: `web/` 폴더. `localhost:5173`. 유저 전용.
- **Command Domain (Dashboard)**: `dashboard/` 폴더 (신규). `localhost:9999`. AI 지휘 전용.
- **API Bridge**: `data/agents/dashboard_api.py`. Port 5000 (Socket.IO). 모든 에이전트와 두 세계를 잇는 통로.

### **📂 2. 모듈화 사령부 UI (Dashboard Modular UI)**
- **상단: AI Status Hub (Cards)**
    - `MainCommandCard`: 메인 AI 전용 대형 카드. [실시간 작업 로그 + 명령 입력창].
    - `AgentStatusCards`: 4인 에이전트(Watcher, Guardian, Debater, Inspector) 소형 카드. [실시간 활동 상태 + 상태 점멸등].
- **하단: Unified Council Feed (Main)**
    - `CouncilFeed`: 6인(파트너+5 AI) 통합 실시간 토론창.

### **⚙️ 3. 파일 구조 모듈화 (Dashboard Component Tree)**
```text
dashboard/
└── src/
    ├── components/
    │   ├── Layout/           # 전체 그리드 레이아웃
    │   ├── Status/           # 에이전트 상태 카드 모듈
    │   ├── Command/          # 메인 AI 로그 및 명령 창 모듈
    │   └── Council/          # 6인 통합 토론 창 모듈
    ├── hooks/                # Socket.IO 통신 훅
    └── store/                # 전역 상태 관리
```

### **🛡️ 4. 가디언 지능형 고도화 (Guardian Intelligence Upgrade)**

- **작업 로그 웹 동기화**: 모든 차단 및 허용 로그를 `MainCommandCard`로 즉시 전송함.

## 📅 [2026-03-09 11:30:00] [누적] 사령부 통합 UI 혁신 및 실시간 신경망 (Supreme V11.0)

### **🏗️ 1. 아프리카TV 스타일 통합 UI (AfreecaTV Style Layout)**
- **목적**: 정보 밀도 극대화 및 사령관(파트너)의 지휘 편의성 확보.
- **구조**: `CouncilFeed.tsx`를 Grid 레이아웃(1fr 320px)으로 전면 개편.
    - **Left Section (80%)**: **Unified Chat Feed**. 대형 메시지 영역으로 가독성 확보.
    - **Right Top Section**: **Council Members**. 사령부 명단 및 실시간 접속/활동 상태 표시.
    - **Right Bottom Section**: **Command Input**. 전용 명령 입력창(Textarea) 배치. `Enter` 키 전송 지원.

### **📡 2. 실시간 신경망 브릿지 (Neural Bridge Architecture)**
- **CLI <-> Dashboard 양방향 동기화**: 터미널의 메인 AI와 웹 대시보드를 물리적으로 연결.
- **REST API 확장**: `dashboard_api.py`에 고속 통신 엔드포인트 추가.
    - `/api/log` (POST): CLI의 작업 내역을 대시보드 'MAIN AI COMMANDER' 로그로 실시간 전송.
    - `/api/chat` (POST): CLI의 답변 내용을 대시보드 토론창으로 실시간 전송.
- **Persistence Layer**: 모든 대화는 `live_chat.md`에 Literal하게 기록되며, 페이지 새로고침 시 초기 히스토리 동기화에 사용됨.

### **⚙️ 3. 데이터 동기화 메커니즘 (Sync Mechanism)**
1. **Dashboard Input** -> **Socket.IO** -> **Backend** -> **live_chat.md (Write)**.
2. **Main AI (CLI)** -> **Read live_chat.md** (Check for new commands) -> **Execute Task**.
3. **Main AI (CLI)** -> **REST API (/api/log, /api/chat)** -> **Dashboard (UI Update)**.
